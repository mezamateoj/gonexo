# AGENTS.md

## Package manager

pnpm only. Never use npm or yarn.

## Non-discoverable commands

### Typechecking (both apps)
Never run `tsc --noEmit` directly — always use the script, which runs `wrangler types` first to regenerate bindings:
```
pnpm run typecheck          # inside apps/back or apps/front
pnpm check:types            # root — runs both
```

### Backend DB workflow
After any schema change in `apps/back/src/db/schema.ts`, run in this order:
```
pnpm run db:local           # = drizzle-kit generate + wrangler d1 migrations apply --local
pnpm run dev                # start worker after migration
```
`db:local` writes SQL to `drizzle/` and applies it to the local D1 store under `.wrangler/`. Skipping it leaves the local DB out of sync — types will pass but queries will fail at runtime.

To migrate production D1:
```
pnpm drizzle-kit migrate    # uses d1-http via CLOUDFLARE_* env vars in .env
```

### Root-level dev/deploy
```
pnpm dev:back / pnpm dev:front
pnpm deploy:back / pnpm deploy:front
```

## Landmines

### `auth:generate` overwrites schema.ts
`apps/back/package.json` has `auth:generate` which calls `better-auth generate` and **writes directly to `src/db/schema.ts`**, clobbering all domain tables. Do not run it. Better Auth tables are already committed; add domain tables manually.

### TanStack Router route tree is auto-generated
`apps/front/src/routeTree.gen.ts` is generated by Vite at dev/build time. Never edit it manually.

### CORS allowlist is hardcoded
`apps/back/src/index.ts` — `allowedOrigins` must be updated manually when adding a new front-end origin. There is no env var for this yet.

### Linter is oxlint, not ESLint
Both apps use `oxlint`. Do not add ESLint config or suggest ESLint-specific fixes.

## Hono + Cloudflare Workers pattern

Always type the app with both `Bindings` and `Variables`:
```ts
const app = new Hono<{ Bindings: Bindings; Variables: Variables }>()
```
`Bindings` comes from `src/binding.ts` (which re-exports from the generated `worker-configuration.d.ts`). Regenerate with `pnpm run generate-types` when `wrangler.jsonc` bindings change.

Use `requireAuth` middleware (already exported from `src/index.ts`) on any protected route.
